# Домашнее задание

## Чем лучше бустить? Тестируем алгоритмы бустинга в бою.

### Цель:
В этом домашнем задании вам предстоит провести детективную работу и узнать, какой же алгоритм бустинга работает лучше всего (конечно, применительно к конкретной задаче).

### Описание / Пошаговая инструкция выполнения домашнего задания:

#### Часть 1. EDA

- Выберите любой интересующий вас датасет по классификации или регрессии (можно взять из репозитория [UCI ML Repository](https://archive.ics.uci.edu/ml/datasets.php), также неплохие и востребованные на практике варианты:
  - предсказание оттока пользователей – [Telco Customer Churn](https://www.kaggle.com/blastchar/telco-customer-churn)
  - предсказание Customer Lifetime Value (CLV или LTV) – [IBM Watson Marketing Customer Value Data](https://www.kaggle.com/pankajjsh06/ibm-watson-marketing-customer-value-data)
- По выбранному датасету проведите **EDA**, познакомьтесь с признаками, посмотрите зависимости и т.д.

#### Часть 2. Preprocessing & Feature Engineering

Хотя цель этого задания – посмотреть на работу алгоритмов, тем не менее пропускать препроцессинг нельзя :)

- Переведите категориальные переменные в уникальные лейблы при помощи `LabelEncoder`
- Попробуйте добавить новые переменные и выкинуть лишние
- Разбейте данные на **train-test**

#### Часть 3. Who's the mightiest of them all?

1. Постройте **4 варианта градиентного бустинга**, используя значения гиперпараметров *«из коробки»*:
   - реализация из `sklearn`
   - `XGBoost`
   - `CatBoost`
   - `LightGBM`
2. Проверьте качество на отложенной выборке. Кто пока лидирует?
3. Теперь проведите **настройку гиперпараметров** моделей на кросс-валидации (можно настраивать только самые основные гиперпараметры: число итераций бустинга, `max_features`, `subsample` и т.д.).
4. Снова проверьте качество уже настроенных моделей. **Кто в итоге победил?**

### Критерии оценки

| Что оценивается | Баллы |
|----------------|-------|
| EDA для выбранного датасета | 1 |
| Preprocessing | 1 |
| Построение моделей из коробки и проверка качества | 4 |
| Настройка гиперпараметров моделей и проверка качества | 4 |
